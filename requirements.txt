#
# Upstream pygame has no wheels for Python 3.13+ yet, and building it from source needs
# SDL headers most people do not have - so `pip install pygame` simply fails there.
# pygame-ce is the maintained community fork: it ships current wheels and is a drop-in
# replacement (still `import pygame`).
#
# These environment markers pick the right one automatically, so this file just works on
# any supported Python. Do not "simplify" them back to a bare `pygame`.
